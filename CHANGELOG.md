# История изменений

Формат основан на [Keep a Changelog](https://keepachangelog.com/ru/1.1.0/).

## [3.9.1] — 2026-05-31

### Исправлено
- **Группы:** напоминание считается доставленным только после отправки в чат (не в личку)
- **Группы:** fallback plain text, если HTML с упоминанием не прошёл
- **Группы:** быстрый retry (30 с) при ошибке доставки в чат
- **Планировщик:** авто-repair каждые 3 мин (пропавшие jobs после рестарта)
- **Права:** предупреждение при создании, если бот не может писать в чат
- **БД:** `next_run_at` хранится как naive local wall clock

## [3.9.0] — 2026-05-31

### Группы и каналы
- **`chat_ctx`:** тип чата (private / group / channel), метки и тексты
- **Каналы:** welcome, `/help`, scoped команды, `/remind` hint
- **Группы:** контекстный `/start` и `/help`, укороченное меню команд
- **Collective UX:** суффиксы «в группе/канале», confirm prefix
- **Голос в группах:** тот же mention-фильтр, что и текст
- План: `docs/plans/groups-channels.md`

## [3.8.1] — 2026-05-31

### Деплой
- Скрипт `setup_github_deploy.ps1` / `.sh` — одна команда для GitHub Secrets
- CI: fail если secrets не настроены; partial mode (Telegram без Wisp API)
- Auto-update: проверка при старте + интервал **1 мин** (было 3)
- `wisp_restart.sh`: fallback Accept header Pterodactyl

## [3.8.0] — 2026-05-31

### Добавлено
- **`/health`:** авто-перепланирование просроченных и напоминаний без задачи в планировщике

### Улучшено
- **Рефакторинг:** общие модули `admin_access`, `deploy_info`, `health_status`, `reminder_create`
- **Handlers:** `menu_keyboard_for_chat` везде вместо `main_menu_keyboard` в группах
- **`is_group_chat`:** единый источник в `timezone_ctx`
- **README:** актуальная версия и команды

## [3.7.0] — 2026-05-31

### Улучшено
- **Группы:** если личка недоступна при срабатывании — подсказка в группе создателю
- **Дубликаты:** один текст в разное время больше не блокируется
- **Редактирование:** batch-фразы (несколько времён/дней) — замена одного на несколько
- **NLP:** «через месяц», «15 июня в 10:00»; LLM «через …» → время от подтверждения

## [3.6.0] — 2026-05-31

### Улучшено
- **Группы:** после создания — подсказка про кнопки в личке; на confirm — «сработает в группе»
- **«Через N мин»:** в карточках показывается «от подтверждения», а не фиксированная дата
- **Batch weekly:** заголовок «по одному на время» при нескольких еженедельных напоминаниях
- **Меню:** reply-клавиатура не показывается в группах
- **`/health`:** просроченные напоминания с учётом timezone; строка deploy sha
- **`/restart`:** корректный перезапуск polling (как при автообновлении)

## [3.5.11] — 2026-05-31

### Изменено
- **Группы:** напоминание сначала в группу, затем в личку с кнопками

## [3.5.10] — 2026-05-31

### Исправлено
- **Планировщик:** SQLite хранит время без tz — после перезапуска бота напоминания сдвигались на 3 ч (UTC vs Москва)
- **Группы:** сначала личка создателю, потом группа; `next_run` очищается только после успешной отправки

## [3.5.9] — 2026-05-31

### Добавлено
- **`/update`** (админ) — принудительное обновление с GitHub + перезапуск

### Исправлено
- Автообновление: если `git pull` не работает — повторный clone как в `start.sh`

## [3.5.8] — 2026-05-31

### Исправлено
- **`/remind`:** фраза из аргументов команды, а не весь текст сообщения — текст задачи без `/remind@бот`
- **«через N минут»:** время считается при подтверждении, а не при разборе — уведомление не теряется из-за задержки

## [3.5.7] — 2026-05-31

### Улучшено
- **Еженедельные фразы:** «еженедельно», «каждый вторник, среду и пятницу», время до дней («в 10 и 16 по пн, ср»)
- **Списки часов:** «в 10, 16 и 21» → три времени без минут
- Текст задачи не включает дни недели и слово «еженедельно»

## [3.5.6] — 2026-05-31

### Исправлено
- **Группы / batch:** после создания показывается «вт, ср, пт, сб в 10:55», а не только дата следующего срабатывания
- Парсер дней недели: списки «во вторник, среду…» и несколько времён через запятую

## [3.5.5] — 2026-05-31

### Добавлено
- **Команда `/remind@бот`** — надёжный способ создать напоминание в группе (Privacy Mode не блокирует команды)

### Изменено
- Подсказки в группах: `/remind` как основной способ; @бот — только из списка Telegram

## [3.5.4] — 2026-05-31

### Исправлено
- Старт: сброс webhook перед polling, лог `@username`, проверка `BOT_TOKEN` в `start.sh`

## [3.5.3] — 2026-05-31

### Исправлено
- **CI/deploy:** lint-ошибки блокировали деплой на Wispbyte с v3.5.1
- **Группы:** логирование, обработка ошибок, без reply-клавиатуры

## [3.5.2] — 2026-05-31

### Изменено
- **Группы:** в подсказках явно — каждую фразу через `@бот`; приветствие с реальным @username

## [3.5.1] — 2026-05-31

### Исправлено
- **Группы:** `@бот` больше не считается упоминанием пользователя
- **«вт, ср, пт, сб в 10:55, 16:10 и 21:00»** — несколько времён → несколько напоминаний

## [3.5.0] — 2026-05-31

### Исправлено
- **«каждый вторник в 10»** — «вт» больше не вырезается из «вторник», задача не ломается
- **«завтра в половине четвёртого»** → 3:30, с **«дня»** → 15:30

### Добавлено
- **«через три-четыре часа»** — словами, берётся большее значение

## [3.4.0] — 2026-05-31

### Добавлено
- **«к обеду»** → 13:00, **«к вечеру»** → 20:00, **«к утру»** → 9:00
- При ошибке парсинга голоса показывается **🎤 Распознано: …**
- Пример «🎤 К обеду» в меню

### Изменено
- Groq prompt дополнен фразами «к обеду», «к вечеру»
- STT: «ко обед», «к вечер» → нормализация

## [3.3.0] — 2026-05-31

### Добавлено
- **«через два часа»**, **«через три минуты»**, **«каждые два часа»** — словами
- Groq Whisper **prompt** для русских напоминаний
- Обработка **аудиофайлов** (не только голос и кружочек)
- STT: «запомни» → «напомни», «ё» → «е», проверка короткого текста

### Изменено
- Пример «🎤 2 часа дня» в меню примеров

## [3.2.0] — 2026-05-30

### Добавлено
- **STT cleanup v2**: «14 00» → «14:00», filler-слова, «напомни мне»
- Словесные часы **13–20**, «четырнадцать ноль ноль»
- **«завтра утром/днём/вечером»** без часа → 9:00 / 14:00 / 20:00
- «полтора дня» → 13:30; подсказки для голоса при ошибке парсинга
- Логирование STT-бэкенда (Groq / Whisper / Yandex)

### Изменено
- Онбординг и /help — советы по голосовым фразам
- LLM-fallback: примеры голосовых фраз в промпте

- Словесные часы: один…двенадцать, «в два», «три часа дня»
- Формы **днём / утром / вечером / ночью**; защита от fallback 9:00

## [3.1.2] — 2026-05-30

### Исправлено
- «в 2 часа дня» → **14:00**, не 02:00; «часа дня» не попадает в текст задачи
- Поддержка **утра / дня / вечера / ночи**, «в 2 дня», полдень, полночь

## [3.1.1] — 2026-05-30

- Префиксы: напомню, напомним, запятая после «напомню,»

## [3.1.0] — 2026-05-30

### Добавлено
- **Groq Whisper STT** — распознавание голоса и кружочков через облако (при `GROQ_API_KEY`)
- Лимиты длительности: голос до 120 с, кружочек до 60 с
- `/sysinfo`: статус ffmpeg и цепочка STT

### Изменено
- Голос и кружочек: единый пайплайн, отдельные подписи «🎤» / «🔵»
- `start.sh` пытается установить ffmpeg на сервере
- Fallback: Groq → локальный Whisper → Yandex
