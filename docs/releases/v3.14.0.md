# v3.14.0

## Native scheduled posts в канале

Разовые напоминания в канале публикуются через **Telegram `schedule_date`** — пост уходит в канал от имени Telegram, без риска пропуска из‑за APScheduler.

### Поведение
- При создании разового напоминания в канале бот ставит **scheduled message** в канал
- В момент срабатывания APScheduler шлёт только **личку** создателю (кнопки Отложить / Готово)
- Snooze, done, delete, pause, clear, edit — **отменяют или пересоздают** scheduled post
- После рестарта `restore` / `repair` восстанавливают и APScheduler, и Telegram schedule

### UX
- Подсказка при добавлении бота в **discussion group** без бота в канале
