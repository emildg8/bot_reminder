# v3.15.2

## Wispbyte restart fix

### Проблема
После auto-update бот вызывал `os.execv` — на Wispbyte это конфликтовало с панелью (KeyboardInterrupt при import aiogram, exit 130, crash loop).

### Решение
- Чистый shutdown → `sys.exit(0)` → панель перезапускает startup-команду
- Stale `data/bot.lock` от упавшего pid удаляется при старте
- Живой pid в lock → выход (нет двух polling одновременно)
