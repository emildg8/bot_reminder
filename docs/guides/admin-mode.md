# Режим администратора / пользователя

Для ID из `ADMIN_TELEGRAM_IDS` (и встроенных в `bot/config.py`).

## Зачем

Проверять бота **глазами пользователя**: лимиты Free, отсутствие `/health`, права в группах — без второго аккаунта.

## Режим

| Команда / кнопка | Действие |
|------------------|----------|
| `/adminmode` | Статус, переключение admin / user |
| `/admin` | Панель (полная в admin-mode, урезанная в user-mode) |
| **👤 / 🛠** | Reply-кнопка режима внизу |
| **⋯ Ещё → 🎛 Админ** | Панель из меню |

**Где видно:** `/ping`, `/status`, `/help`, `/start`, меню команд Telegram.

В **user-mode** скрыты ops-команды; остаются `/adminmode` и ops-уведомления при падении бота.

## Пользователи

| Команда | Действие |
|---------|----------|
| `/userinfo ID` или **ответ** | Карточка: TZ, напоминания, чаты, Pro |
| `/userfind 123456` | Поиск ID (≥3 цифр), кнопки → userinfo |
| `/grantpro` · `/revokepro` | ID или **ответ** |
| **Недавние** (панель) | Список + inline-кнопки |

## Рассылка

| Команда | Действие |
|---------|----------|
| `/broadcast текст` | Превью, фильтры, «Себе», «Отправить» |
| `/broadcast активные …` | Только с активными напоминаниями |
| `/broadcast pro …` · `free …` | Аудитория Pro / Free |
| `/broadcast test …` | Пример себе |
| `/broadcast да …` | Сразу (до 100) |

## Ops

| Команда | Действие |
|---------|----------|
| `/adminstats` | Статистика: типы, группы, планировщик |
| `/adminlog` | Журнал действий (in-memory, сброс при рестарте) |
| `/admins` | Список админов («← ты») |

## Технически

- БД: `users.admin_tools_enabled` (миграция `20260601_0002`)
- `bot/services/admin_access.py` — права и кэш режима
- `bot/services/admin_mode.py` — меню команд, клавиатуры
- `bot/services/admin_panel.py` — панель, userinfo, broadcast
- `bot/services/admin_audit.py` — журнал действий
