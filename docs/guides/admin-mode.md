# Режим администратора / пользователя

Для ID из `ADMIN_TELEGRAM_IDS` (и встроенных в `bot/config.py`).

## Зачем

Проверять бота **глазами пользователя**: лимиты Free, отсутствие `/health`, права в группах — без второго аккаунта.

## Команды

| Команда / кнопка | Действие |
|------------------|----------|
| `/admin` | **Панель** — все инструменты кнопками |
| `/adminmode` | Режим admin/user + быстрые кнопки |
| `/userinfo ID` или **ответ** + `/userinfo` | Карточка, чаты, ближайшее напоминание, кнопки |
| `/admins` | Список админов («← ты») |
| `/grantpro` · `/revokepro` | ID или **ответ** на сообщение |
| `/broadcast текст` | Превью, фильтры, «Себе», «Отправить» |
| `/broadcast активные\|pro\|free текст` | Аудитория по фильтру |
| `/broadcast test текст` | Пример рассылки себе |
| `/broadcast да текст` | Рассылка сразу (до 100) |
| **Недавние** | Inline-кнопки → userinfo |
| `/userfind 123456` | Поиск ID по фрагменту (≥3 цифр) |
| `/adminstats` | Расширенная статистика |
| `/adminlog` | Журнал действий (in-memory) |
| `/admin` в user-mode | Урезанная панель + включить 🛠 |
| **⋯ Ещё → 🎛 Админ** | Панель из меню |
| **👤 / 🛠** | Reply-кнопка режима внизу |

## Где видно режим

- `/ping` — `🛠 admin` или `👤 user-test`
- `/status` — строка режима (+ лимиты Free в user-mode)
- `/help` и `/start` — блок «Админ бота»
- Меню команд Telegram — `/health` только в admin-режиме

## Режим пользователя

Скрыты: `/health`, `/sysinfo`, `/update`, `/setavatar`, `/grantpro` (из меню команд).

Остаются: `/adminmode`, ops-уведомления в Telegram при падении бота.

## Технически

- БД: `users.admin_tools_enabled` (по умолчанию `true`)
- Код: `bot/services/admin_access.py`, `bot/services/admin_mode.py`
